# ProjectDSDE_election2026

Data Science for Thailand Election 2026
Course: 2110446 Data Science and Data Engineering
Constituency: ชัยภูมิ เขต 2
Polling stations: 341

## Project Overview

This project converts official Election Commission of Thailand PDF documents into structured election datasets, validates the extracted values, analyzes the results, and presents them through a Streamlit dashboard.

The selected case study is Chaiyaphum Constituency 2. The workflow covers:

1. Reference data and source-PDF coverage preparation
2. OCR extraction from raw PDF pages
3. Data cleaning, validation, and imputation
4. Exploratory analysis and version comparison
5. Dashboard presentation

`data/` is intentionally ignored by Git because the raw PDFs, rendered images, OCR cache, and generated tables are large local artifacts. A fresh checkout needs the data folder restored from the project artifact package or regenerated by following the notebook order below.

## Setup

Create a Python virtual environment and install dependencies:

```bash
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip
.venv/bin/python -m pip install -r requirements.txt
```

OCR preprocessing uses `pdf2image`, which requires Poppler:

```bash
brew install poppler
```

Typhoon OCR requires an API key only when OCR requests are made:

```bash
cp .env.example .env
# edit .env and set TYPHOON_API_KEY=...
```

Do not commit `.env`.

## Run Order

1. Prepare reference data and source manifest:

```bash
jupyter notebook notebooks/01_download_data.ipynb
```

2. Run a small OCR smoke test:

```bash
.venv/bin/python run_batch.py run --limit-pdfs 1
```

3. Run full OCR after checking the smoke-test cache and checkpoint output:

```bash
.venv/bin/python run_batch.py run
```

4. Aggregate checkpoints and generate QA output:

```bash
.venv/bin/python run_batch.py aggregate
.venv/bin/python run_batch.py qa
```

5. Run the cleaning, EDA, and comparison notebooks in order:

```text
notebooks/03_cleaning_validation.ipynb
notebooks/04_eda_analysis.ipynb
notebooks/05_comparison_analysis.ipynb
```

6. Open the dashboard:

```bash
.venv/bin/streamlit run notebooks/dashboard.py
```

## OCR Run Safety

- `notebooks/02_ocr_extraction.ipynb` defaults to a one-PDF smoke test.
- To run the full OCR batch from the notebook, set `RUN_FULL_OCR = True`.
- `OCR_FORCE = False` is the default so existing OCR cache and checkpoints are not overwritten accidentally.
- Missing `TYPHOON_API_KEY` raises an error only when an OCR request is attempted.

## Validation

Run checks that do not call OCR or create new data outputs:

```bash
.venv/bin/python -m pytest
.venv/bin/python -m py_compile run_batch.py notebooks/dashboard.py
```

The current test suite checks that Python entry points compile, notebook code cells compile, and notebook 02 does not run a full OCR batch by default.

## Git Hygiene

The generated `data/` folder stays out of Git. If a data sample or review file must be submitted through Git, confirm the exact file first and add it intentionally.
